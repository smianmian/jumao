# 参与贡献橘猫

谢谢你愿意让橘猫对非技术用户更有用。

橘猫不是“自动生成完整 App”的承诺。它是一套普通人能看懂的产品落地框架，用来把想法整理成 AI agent 能执行的任务，并且保留边界、安全和证据。

## 什么是好的贡献

- 语言要让非技术用户看得懂。
- 保留核心流程：想法、边界、页面状态、数据安全、证据、任务包。
- 不要让 CLI 直接调用 AI API。
- 不要默认加入 API Key、云账号、付费、托管服务或外部平台依赖。
- 没有证据，不要声称已经上线、审核通过、部署完成或任务完成。
- 改用户可见内容时，中英文文档要尽量同步。

## 提交 PR 前

请先运行：

```bash
npm run check
npm pack --dry-run
git status --short
```

PR 里请写清楚：

- 改了什么。
- 为什么这对非技术用户有帮助。
- 没有改什么。
- 用什么证据验证过。

## 写作风格

- 优先用普通话，不要堆工程术语。
- 必须用技术词时，顺手解释一句。
- 不要承诺橘猫能自己做出完整产品。
- 风险动作要写明白：真实用户、生产数据、付费、上线、审核、外部账号，都需要人工确认。
