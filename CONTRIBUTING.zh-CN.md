# 参与贡献橘猫

谢谢你愿意一起把橘猫变得更好用。

橘猫服务的是不太会写代码、但想把产品做出来的人。好的改动应该让他们更容易写清楚想法、边界、数据和验收证据。

## 什么是好的贡献

- 语言要让不懂代码的人也看得懂。
- 保留核心流程：想法、边界、页面状态、数据安全、证据、任务包。
- 不要让 CLI 直接调用 AI API。
- 不要默认加入 API Key、云账号、付费、托管服务或外部平台依赖。
- 没有证据，不要声称已经上线、审核通过、部署完成或任务完成。
- 改用户可见内容时，中英文文档要尽量同步。

## 提交 PR 前

请先运行：

```bash
npm run check
npm pack --dry-run
git status --short
```

PR 里请写清楚：

- 改了什么。
- 为什么这对普通用户有帮助。
- 没有改什么。
- 用什么证据验证过。

## 写作风格

- 优先用普通话，不要堆工程术语。
- 必须用技术词时，顺手解释一句。
- 不要承诺橘猫能自己做出完整产品。
- 风险动作要写明白：真实用户、生产数据、付费、上线、审核、外部账号，都需要人工确认。
