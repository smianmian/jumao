# 安全政策

橘猫是本地 CLI 和文档工具包。默认不调用 AI API，不读取 API Key，不保存密钥，也不连接托管服务。

## 支持版本

当前 `main` 分支和最新发布版本会被支持。

## 报告安全问题

请不要用公开 issue 报告安全问题。

如果仓库已经托管在 GitHub，优先使用 GitHub private vulnerability reporting。如果还没有开启，请通过仓库所有者账号私下联系维护者。

请包含：

- 发生了什么。
- 尽量小的复现步骤。
- 是否涉及密钥、用户数据、支付数据、生产数据或外部账号。
- 不暴露密钥的日志或截图。

## 范围内问题

安全相关问题包括：

- 意外读取或写入 API Key、token 或 `.env` 文件。
- CLI 未经用户明确操作就向网络服务发送数据。
- 生成的指引让 AI agent 在没有确认时触碰生产数据、付费、上线、审核或外部账号。
- npm 包内容意外包含密钥或私有项目文件。

## 范围外问题

- 要求橘猫直接加入 AI API 调用。
- 使用橘猫做出来的其他 App 自身的问题。
- 已经公开泄露密钥后，要求维护者从第三方系统清理。

## 维护者检查

发布前运行：

```bash
npm run check
npm pack --dry-run
git status --short
```

检查包内容，确认不包含密钥、本地工作目录或私有证据。
