# Jumao 内置 Agent 是什么

你不用雇 44 个人。
Jumao 内置了 44 个责任单元 Agent。
它们像一组检查员，帮你在让 Codex 写代码之前，看清楚项目缺什么、哪些事可以先不做、哪些事不能乱做。

这些 Agent 不会替你写代码。
它们只做三件事：
- 用人话解释你可能不知道的坑
- 根据你的回答判断哪些风险会出现
- 告诉 Codex 哪些事情现在不能做

## 怎么读这份说明

每个 Agent 都是一种检查视角，不是一个要你招聘的人。

你只需要回答生活化问题，比如：
- 这个项目是自己玩，还是准备上线？
- 要不要别人登录？
- 要不要收费？
- 要不要上 App Store？
- 用户换手机后，数据要不要还在？
- 有没有健康、金融、儿童、定位、通讯录这类敏感数据？

Jumao 会在后台判断哪些 Agent 该出现。

表格里每一行只回答三件事：
- 它帮你判断什么。
- 什么情况会触发它。
- 它会拦住 Codex 乱做什么。

如果你看不懂某个英文词，也不用停下来查。
先看中文解释，Jumao 会把专业问题翻译成人话。

## 方向与主体 Agent 组

| Agent | 它帮你判断什么 | 什么时候会出现 | 它会拦住 Codex 乱做什么 |
|---|---|---|---|
| 项目负责人 / 创始人 Agent | 判断自用、试用、上线还是公司项目 | 你说要上线、收费、公司主体时 | 主体和边界没定前，不写业务代码 |
| 公司注册 / 行政负责人 Agent | 判断个人名义还是公司名义 | 你说公司产品或大陆上线时 | 主体资料不清前，不写上线结论 |
| 知识产权 / 商标 Agent | 判断名字、图标、素材是否稳妥 | 你准备公开上线或做品牌时 | 没有授权证据，不说可商用 |
| 软件著作权 / 资质留存负责人 Agent | 判断是否要留版本和资质材料 | 你把项目当商业产品时 | 没有材料前，不说资质已备好 |
| 采购 / 合同 / 供应商负责人 Agent | 判断外部服务要留什么记录 | 你买服务、找外包、接供应商时 | 供应商边界不清，不接关键服务 |

## 产品与设计 Agent 组

| Agent | 它帮你判断什么 | 什么时候会出现 | 它会拦住 Codex 乱做什么 |
|---|---|---|---|
| 产品经理 Agent | 判断第一版先做什么不做什么 | 所有项目都会出现 | 范围没写清前，不自己加功能 |
| UI / UX 设计师 Agent | 判断每一步用户看到什么 | 你要做页面、流程、原型时 | 状态没写清前，不只做顺利路径 |
| 品牌 / 文案 Agent | 判断对外说法是否清楚克制 | 你要上线、收费、写商店文案时 | 没有证据前，不写夸大承诺 |
| 用户研究 / 市场定位负责人 Agent | 判断第一批用户是谁 | 你还在想法、原型、上线前 | 用户没说清前，不写成所有人用 |
| 设计系统 / Design QA 负责人 Agent | 判断界面风格是否一致 | 页面变多或准备内测时 | 规则没定前，不批量写正式页面 |
| 无障碍 / 可访问性负责人 Agent | 判断更多用户能不能顺利用 | 你准备公开上线或上架时 | 不让颜色成为唯一状态提示 |
| 文档 / 交付物负责人 Agent | 判断给 AI 的说明是否清楚 | 你准备把任务交给 Codex 时 | 任务包不清前，不直接改代码 |

## 给 Codex 的硬规则

- 没有 DATA_GOVERNANCE_REGISTER.md，不得新增数据库字段。
- 没有 SDK_VENDOR_REGISTER.md，不得引入第三方 SDK。
- 没有 HEALTH_CLAIMS_APPROVAL_LOG.md，不得新增健康结论、推送文案、报告文案。
- 没有 IAP_REVENUE_OPS_CHECKLIST.md，不得接入 StoreKit 生产订阅。
- 没有 CLOUD_IAM_SECRETS_BACKUP_SPEC.md，不得部署生产环境。
- 没有 RELEASE_MANAGER_CHECKLIST.md，不得提交 TestFlight 或 App Store 审核包。
- 没有 SUPPORT_REFUND_DELETION_PLAYBOOK.md，不得上线带登录和订阅的版本。
- 没有 SCREEN_INVENTORY.md 和 STATE_MATRIX.md，不得写 SwiftUI 页面。
- 没有 ORG_ROLE_OWNER_MATRIX.md，不得开始业务代码。
