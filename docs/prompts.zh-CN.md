# AI Agent 提示词

这些提示词适合复制给 Codex、Claude Code、Cursor 或其他 AI 编程工具。先让 AI 读橘猫任务包，再让它动代码。

## 第一次交给 AI

```text
请先读取这个项目里的 AGENTS.md、product/ 和 proof/。
不要急着写代码。

请用普通人能懂的话告诉我：
1. 这个产品给谁用；
2. 首版只做什么；
3. 首版明确不做什么；
4. 目前缺哪些信息；
5. 下一步最小安全动作是什么。

如果你发现任何会影响真实用户、生产数据、付费、上线、审核或外部账号的动作，先停下来问我。
```

## 让 AI 开始实现一个小任务

```text
现在只实现一个小任务：
<写下这次任务>

请先说明：
1. 这次改动对应哪个首版目标；
2. 会改哪些文件；
3. 不会改哪些范围；
4. 准备用什么命令或证据验证。

我确认后再开始改代码。
```

## 让 AI 自查是否跑偏

```text
请对照橘猫任务包自查：
1. 有没有扩大首版范围；
2. 有没有加入任务包明确不做的功能；
3. 有没有碰真实用户、生产数据、付费、上线、审核或外部账号；
4. 有没有留下测试、截图、日志或人工验收证据；
5. 还有什么不能说完成。
```

## 让 AI 写完成证据

```text
请把本轮结果写进 proof/release-proof.zh-CN.md。

必须包含：
- 本轮改了什么；
- 没有改什么；
- 验证命令和结果；
- 还不能说完成的事；
- 下一步建议。

没有证据的地方不要写成已完成。
```

## 交给 Codex 的推荐开场

```text
你现在是橘猫项目落地助手。
请先读 AGENTS.md 和 jumao-task-pack.md。
先总结目标、边界、风险和下一步计划，不要直接改代码。
```

## 交给 Claude Code 的推荐开场

```text
请先读 CLAUDE.md、AGENTS.md 和 product/ 下的材料。
你需要先告诉我产品目标、首版边界、缺口和下一步最小安全动作。
我确认前不要写代码。
```

## 交给 Cursor 的推荐开场

```text
请把 AGENTS.md 当作项目规则。
每次改动前先回答：这次改动对应哪个首版目标？
如果改动会扩大首版范围，请先停下来问我。
```
